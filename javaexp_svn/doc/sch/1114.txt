### 1. 흥미로운 도입
웹 개발의 세계에서 JSP와 DAO의 연동은 마치 레스토랑에서 주문이 요리사에게 전달되는 
과정과 유사합니다. 
손님이 메뉴를 선택하면(사용자가 웹 페이지에서 요청), 웨이터(서블릿)가 주문을 받아 
요리사(DAO)에게 전달합니다. 
요리사는 주방(데이터베이스)에서 음식(데이터)을 준비하여 웨이터를 통해 손님에게 제공합니다. 
이 과정을 통해 웹 애플리케이션의 효율성과 구조화를 실현할 수 있습니다.
### 2. 학습목표
- JSP에서 사용자 요청을 처리하는 방법 이해
- DAO(Data Access Object) 패턴의 중요성 및 구현 방법 학습
- JSP와 DAO 간의 데이터 흐름 이해
- JDBC를 사용한 데이터베이스 연동 기술 습득
### 3. 주요 핵심 키워드와 개념
- **JSP(Java Server Pages)**: 서버 사이드에서 동적인 웹 페이지를 생성하는 기술
- ** request.getParameter("") : 서버단에서 요청값을 받을 때, 처리 내용
- ** form action  name value submit : 클라이언트 단에서 요청값을 전송
- **서블릿(Servlet)**: 클라이언트의 요청을 처리하고, 그 결과를 반환하는 서버 사이드
                   컴포넌트
- **MVC(Model-View-Controller)**: 애플리케이션을 세 부분(모델, 뷰, 컨트롤러)으로 
	분리하는 디자인 패턴

### 6. 실무적용 예시
- **사용자 로그인 기능**
  - 사용자의 로그인 요청을 JSP 페이지에서 받음
  - 요청 정보를 서블릿으로 전달하고, 서블릿은 해당 정보를 DAO에 전달
  - DAO는 JDBC를 사용하여 데이터베이스에서 사용자 정보를 조회
  - 조회 결과를 서블릿을 통해 JSP 페이지에 전달하여 사용자에게 로그인 성공 또는 실패 메시지 표시

- **데이터 등록 및 조회 기능**
  - 사용자로부터 입력받은 데이터를 JSP 페이지에서 처리
  - 서블릿을 통해 DAO에 데이터 전달
  - DAO는 해당 데이터를 데이터베이스에 저장하고, 저장된 데이터의 조회 기능 제공

이 커리큘럼을 통해 JSP와 DAO를 활용한 웹 애플리케이션의 개발 과정을 체계적으로 이해하고 실습해 볼 수 있습니다.
